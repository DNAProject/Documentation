
## 简介

默认情况下，`Restful` 接口监听在 `20334` 端口。

- 使用 `--rest` 选项可以开启客户端的 `Restful` 服务。

```shell
./dnaNode --rest
```

- 使用 `--restport` 选项为客户端指定 `Restful` 监听端口。

```shell
./dnaNode --rest --restport 1024
```

## 接口定义

DNA区块链客户端的 `Restful` 接口定义如下：

- 默认接口监听端口号：`20334`

- 响应参数

| 字段    | 类型   | 描述         |
| :------ | :----- | :----------- |
| Action  | 字符串 | 响应动作名称 |
| Desc    | 字符串 | 响应结果描述 |
| Error   | 整型   | 错误代码     |
| Result  | 对象   | 执行结果     |
| Version | 字符串 | 版本号       |

- 错误码

| Field | Type  | Description      |
| :---- | :---- | :--------------- |
| 0     | int64 | 成功             |
| 41001 | int64 | 无效或超时的会话 |
| 41002 | int64 | 达到服务上限     |
| 41003 | int64 | 不合法的数据格式 |
| 41004 | int64 | 无效的版本号     |
| 42001 | int64 | 无效的方法       |
| 42002 | int64 | 无效的参数       |
| 43001 | int64 | 无效的交易       |
| 43002 | int64 | 无效的资源       |
| 43003 | int64 | 无效的区块       |
| 44001 | int64 | 未知的交易       |
| 44002 | int64 | 未知的资源       |
| 44003 | int64 | 未知的区块       |
| 45001 | int64 | 内部错误         |
| 47001 | int64 | 智能合约执行错误 |

```

```

```json
curl -i http://localhost:20334/api/v1/storage/ff00000000000000000000000000000000000001/0144587c1094f6929ed7362d6328cffff4fb4da2
```
响应：

```json
{
    "Action": "getstorage",
    "Desc": "SUCCESS",
    "Error": 0,
    "Result": "58d15e17628000",
    "Version": "1.0.0"
}
```

<section class = "info">
  <ul>
    <li>请求中的 <code>key</code> 为十六进制字符串形式的合约存储键值。</li>
    <li>响应中的 <code>result</code> 所对应的值为序列化后的十六进制字符串，可以使用 <code>SDK</code> 进行反序列化。</li>
  </ul>
</section>

根据智能合约编译后所得到的 `avm code`，可以使用 `SDK` 生成合约地址。

```python
from DNA.common.address import Address
from DNA.utils.contract_data import ContractDataParser

code = '0000000000000000000000000000000000000004'
contract_address = Address.address_from_vm_code(code).to_hex_str()
```

## 20. getversion

获取当前连接节点的版本信息。

```shell
GET /api/v1/version
```

#### 调用示例

请求：

```json
curl -i http://localhost:20334/api/v1/version
```

响应：

```json
{
    "Action": "getversion",
    "Desc": "SUCCESS",
    "Error": 0,
    "Version": "1.0.0",
    "Result": "0.9"
}
```

## 21. post_raw_tx

向DNA区块链网络发送交易。

如果 `preExec=1`，则交易为预执行。

POST

```
/api/v1/transaction?preExec=0
```

```json
curl  -H "Content-Type: application/json"  -X POST -d '{"Action":"sendrawtransaction", "Version":"1.0.0","Data":"00d00000000080fdcf2b0138c56b6c766b00527ac46c766b51527ac46151c56c766b52527ac46c766b00c31052656749644279507..."}'  http://server:port/api/v1/transaction
```
